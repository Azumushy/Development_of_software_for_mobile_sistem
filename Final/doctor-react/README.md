# Doctor Reviews - Мобильное приложение для просмотра врачей и отзывов

React Native приложение для просмотра информации о врачах, оставления и управления отзывами.

## Функциональность

### 1. Система аутентификации
- Регистрация нового пользователя (email и пароль)
- Подтверждение пароля при регистрации
- Валидация данных на клиенте
- Вход в систему по email/паролю
- Сохранение сессии пользователя
- Выход из системы

### 2. Работа с врачами
- Главный экран со списком врачей
- Поиск врачей по имени/специальности
- Фильтрация по специальности
- Сортировка (по рейтингу, имени, опыту)
- Карточка врача с фото/аватаром, ФИО, специальностью, опытом, рейтингом
- Детальная страница врача с полной информацией
- Список отзывов о враче на детальной странице
- Кнопка "Оставить отзыв"

### 3. Система отзывов
- Просмотр отзывов с пагинацией
- Сортировка отзывов (по дате, рейтингу)
- Рейтинг врача на основе отзывов (автоматический пересчет)
- Добавление отзыва (оценка 1-5 звезд, комментарий, дата посещения)
- Только для авторизованных пользователей
- Редактирование собственных отзывов
- Удаление собственных отзывов с подтверждением

### 4. Личный кабинет
- Просмотр профиля пользователя
- Смена аватара (выбор из галереи)
- История своих отзывов
- Статистика активности (количество отзывов, средняя оценка)
- Быстрый переход к врачу из списка отзывов

## Архитектура

### Структура компонентов
Каждый компонент следует принципу единственной ответственности:
- `Button` - переиспользуемая кнопка
- `Input` - поле ввода с валидацией
- `StarRating` - компонент рейтинга со звездами
- `DoctorCard` - карточка врача
- `SearchBar` - строка поиска
- `FilterBar` - панель фильтров и сортировки
- `ReviewCard` - карточка отзыва
- `ReviewList` - список отзывов

### Структура экранов
- `LoginScreen` - экран входа
- `RegisterScreen` - экран регистрации
- `HomeScreen` - главный экран со списком врачей
- `DoctorDetailScreen` - детальная страница врача
- `AddReviewScreen` - добавление отзыва
- `EditReviewScreen` - редактирование отзыва
- `ProfileScreen` - личный кабинет

### Утилиты
- `AuthContext` - контекст для управления аутентификацией
- `storage.js` - функции для работы с AsyncStorage
- `validation.js` - функции валидации
- `constants.js` - константы приложения

## Установка и запуск

### Требования
- Node.js (версия 14 или выше)
- npm или yarn
- Expo CLI
- Expo Go приложение на мобильном устройстве (для тестирования)

### Установка зависимостей
```bash
cd doctor-react
npm install
```

### Запуск приложения
```bash
npm start
```

Затем:
- Нажмите `i` для запуска на iOS симуляторе
- Нажмите `a` для запуска на Android эмуляторе
- Отсканируйте QR-код в приложении Expo Go на вашем устройстве

## Особенности реализации

1. **Хранение данных**: Используется AsyncStorage для локального хранения данных (врачи, отзывы, пользователи)

2. **Валидация**: Клиентская валидация всех форм ввода

3. **Навигация**: React Navigation с нативной стековой навигацией и табами

4. **Состояние**: React Context для глобального состояния аутентификации

5. **Стилизация**: StyleSheet для всех компонентов, единая цветовая схема через константы

6. **Инициализация данных**: При первом запуске создаются тестовые данные о врачах
